    Workflow Commands Reference

    Complete reference for all 12 ADHD-friendly workflow commands. Designed to reduce decision paralysis and maintain focus.

    Command Categories

    • Core Workflow (7 commands)
    • Task Management (3 commands)
    • Documentation (1 command)

    Core Workflow

    /brainstorm

    Smart ideation with auto-delegation and mode detection.

    # Auto-detect mode from context
/brainstorm

# Specific modes
/brainstorm quick "Feature ideas"           # <1 min, no delegation
/brainstorm "Architecture design"           # <5 min, balanced
/brainstorm thorough "Production strategy"  # <30 min, deep analysis

# Specific topics
/brainstorm feature "User profiles"
/brainstorm architecture "Real-time system"
/brainstorm design "Mobile-first UI"

# Output formats
/brainstorm --format json
/brainstorm --format markdown > brainstorm.md

    Features: - Auto-activating skills (backend/frontend/devops) - Background agent delegation - Multiple output formats - Time budgets enforced

    /focus

    Single-task mode with distraction blocking.

    /focus "Implement OAuth authentication"
/focus "Fix memory leak in worker process"

    Sets: - Clear objective - Success criteria - Time budget (default: 2 hours) - Milestone reminders

    Output: 

    üìç Focus Mode Activated

Milestones: Setup (30min), Implementation (60min), Testing (30min)

    /done

    Session completion with context capture.

    /done
/done "Completed OAuth implementation, tests passing"

    Captures: - Achievements - Blockers encountered - Context for next session - Updates .STATUS automatically

    Prompts for: - What did you complete? - Any blockers? - What's next?

    /recap

    Context restoration when returning to work.

    /recap

    Reviews: - Recent commits (last session) - .STATUS file - Open TODO items - Suggests where to resume

    /stuck

    Unblock helper with guided problem solving.

    /stuck
/stuck "Can't figure out why tests are failing"

    5 Why Analysis: 1. What's the immediate blocker? 2. What have you tried? 3. What's a simpler version? 4. Who/what could help? 5. What would success look like?

    Provides: - Alternative approaches - Simplification strategies - Resource suggestions - Success definition

    /refine

    Prompt optimizer for better AI interactions.

    /refine "make the app faster"

    Improves: - Clarity and specificity - Actionable language - Context inclusion - Expected outcomes

    Example: 

    Original: "make the app faster"

Refined: "Profile the React application to identify rendering bottlenecks. Focus on the UserDashboard component which users report as sluggish. Generate a performance report with specific optimization recommendations."

    Task Management

    /task-status

    Check progress of background tasks.

    /task-status

    Shows: - Active agents - Execution time - Estimated completion - Current status

    /task-output

    View results from completed background task.

    /task-output <task-id>

    Returns: - Formatted synthesis - Agent outputs - Actionable recommendations

    /task-cancel

    Cancel running background task.

    /task-cancel <task-id>

    Actions: - Stops agent execution - Cleans up resources - Reports partial results if available

    Time Budgets

    All commands respect explicit time guarantees:

    Command Mode Time Budget
    /brainstorm quick <60s
    /brainstorm default <5min
    /brainstorm thorough <30min
    /focus - User-defined (default: 2hr)
    /next - <10s
    /done - <30s
    /recap - <30s
    /stuck - <2min
    /refine - <30s

    ADHD-Friendly Features

    Scannable Output

    All commands use visual hierarchy: - Emojis for quick scanning - Clear sections with boxes - Numbered action items - Time estimates visible

    Reduced Decisions

    Commands make decisions FOR you: - /next ‚Üí ONE recommendation (not a list) - /focus ‚Üí Clear objective set - /done ‚Üí Auto-updates status

    Quick Wins

    Always included: - /next ‚Üí Includes <15min alternative - /brainstorm ‚Üí Quick mode available - /stuck ‚Üí Simpler version suggested

    Context Preservation

    Commands help maintain context: - /done ‚Üí Captures session context - /recap ‚Üí Restores context quickly - /focus ‚Üí Sets clear boundaries
